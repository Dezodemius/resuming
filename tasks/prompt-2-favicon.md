# Задача: заменить favicon на новый зелёный logo-mark

Сайт **Резюмирую.рф** пережил редизайн: в шапке приложения теперь зелёный знак `.logo-mark` — белый контурный «документ» на зелёном градиенте со скруглённым квадратом. А favicon в `static/` остался старый (тёмно-синий документ с молнией). Нужно перегенерировать весь favicon-сет из нового знака.

## Эталон дизайна

Знак описан в `static/app.css:71-72`:
- квадрат 30×30 с радиусом скругления 9px (т.е. **30% стороны**);
- фон: линейный градиент 140° от `#34d399` (верхний левый край) к `#10b981` (нижний правый);
- иконка внутри — SVG-документ, штрих цвета `#04130d`, толщина 2.4 при viewBox 24, скруглённые концы и стыки:
  - контур: `M5 4 h10 l4 4 v12 H5 z` (прямоугольник с загнутым верхне-правым углом);
  - строки текста: `M9 11 h6` и `M9 15 h4`;
- иконка занимает ~57% стороны квадрата (17px из 30), по центру.

## Шаг 1. SVG-favicon

Создай `static/favicon.svg` — точная векторная версия знака:

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 48 48">
  <defs>
    <linearGradient id="g" x1="0" y1="0" x2="1" y2="1">
      <stop offset="0" stop-color="#34d399"/>
      <stop offset="1" stop-color="#10b981"/>
    </linearGradient>
  </defs>
  <rect width="48" height="48" rx="14" fill="url(#g)"/>
  <g transform="translate(12 12) scale(1.0)" fill="none" stroke="#04130d"
     stroke-width="2.4" stroke-linecap="round" stroke-linejoin="round">
    <path d="M5 4h10l4 4v12H5z"/>
    <path d="M9 11h6M9 15h4"/>
  </g>
</svg>
```

Подгони `translate`/`scale` так, чтобы иконка 24×24 была по центру и занимала ~57–60% стороны (при 48px: масштаб ≈1.15, смещение ≈(10.2, 10.2)). Посмотри результат сам: открой SVG через Read после конвертации в PNG на шаге 2 и убедись, что композиция совпадает с эталоном.

## Шаг 2. Скрипт растеризации (Pillow, без cairo)

Внешние SVG-растеризаторы на Windows капризны, поэтому нарисуй растр геометрически. Создай `tools/gen_favicons.py`:

1. `pip install pillow` (только в локальное окружение; **в requirements.txt НЕ добавлять** — это dev-инструмент).
2. Рисуй мастер-изображение 1024×1024 (RGBA, прозрачный фон):
   - **Градиентная подложка**: диагональный градиент `#34d399 → #10b981`. Простой способ: построить массив, где параметр `t = (x + y) / (2 * size)`, и линейно интерполировать каналы (при 1024px попиксельный цикл в Python медленный — построй градиент на 256×256 и отмасштабируй до 1024 через `Image.resize(..., LANCZOS)`).
   - **Скругление**: маска-`rounded_rectangle` с радиусом `0.30 * size`, применить как альфа-канал.
   - **Документ**: масштаб `s = 1024 * 0.58 / 24`, сдвиг, центрирующий поле 24×24. В координатах viewBox:
     - контур — замкнутая ломаная `(5,4) → (15,4) → (19,8) → (19,20) → (5,20) → (5,4)`;
     - линии — `(9,11)–(15,11)` и `(9,15)–(13,15)`;
     - штрих: ширина `2.4 * s`, цвет `#04130d`; рисуй `ImageDraw.line(..., joint="curve")`, а скруглённые концы добавь кружками радиусом `1.2 * s` в конечных точках каждой линии.
   - Рисуй с 4× суперсэмплингом (мастер 4096, потом уменьшение до 1024), чтобы края были гладкими.
3. Из мастера сгенерируй и **перезапиши** файлы в `static/`:
   - `favicon-16x16.png`, `favicon-32x32.png`
   - `android-chrome-192x192.png`, `android-chrome-512x512.png`
   - `apple-touch-icon.png` (180×180) — **без прозрачности и без скругления**: залей весь квадрат градиентом (iOS сам скругляет углы; прозрачные углы дадут чёрные артефакты);
   - `favicon.ico` — мультиразмерный: `img.save("static/favicon.ico", sizes=[(16,16),(32,32),(48,48)])`.

Запусти скрипт. Затем **посмотри результат глазами**: открой `static/android-chrome-512x512.png` через Read и сравни с эталоном (зелёный градиент, тёмный документ по центру, скругления). Если пропорции кривые — поправь и перегенерируй.

## Шаг 3. Подключение в шаблонах

Favicon-ссылки прописаны в 6 шаблонах: `templates/index.html`, `resumes.html`, `pricing.html`, `settings.html`, `resume_edit.html`, `_legal_base.html` (строки ~9–12 в каждом). В каждом:

1. Добавь **первой** строкой: `<link rel="icon" type="image/svg+xml" href="/static/favicon.svg?v=2">`.
2. К существующим ссылкам (ico, 16, 32) допиши `?v=2` — сайт за Cloudflare, без cache-bust старая иконка будет висеть в кеше.
3. Если в шаблоне есть `<link rel="apple-touch-icon" ...>` — тоже `?v=2`; если нет — добавь `<link rel="apple-touch-icon" sizes="180x180" href="/static/apple-touch-icon.png?v=2">`.

## Шаг 4. Проверка и коммит

1. `python -m pytest -q` — тесты должны пройти (они не про favicon, но проверяют, что шаблоны не сломаны).
2. `python -c "import main"` — импорт без ошибок.
3. Убедись, что все 7 файлов в `static/` обновлены (свежая дата изменения) и `favicon.svg` создан.
4. Закоммить и запушь в текущую ветку (подтверждения не нужны). В коммит включи и `tools/gen_favicons.py`:

```
feat(ui): новый favicon — зелёный logo-mark из редизайна

Co-Authored-By: Claude <noreply@anthropic.com>
```

## Чего НЕ делать

- `static/banner.png` не трогай — это логотип для шапок старых страниц, он вне задачи.
- Pillow в `requirements.txt` не добавлять.
- Никаких внешних сервисов-генераторов favicon и скачиваний из интернета.
- Скриншоты из `redesign/` не редактировать.
