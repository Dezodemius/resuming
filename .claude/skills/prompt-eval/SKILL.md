# Prompt-eval — регрессионный прогон промптов через Ollama

Скрипт `.claude/skills/prompt-eval/eval.py` импортирует функции промптов и `call_ai()` прямо из `main.py` (eval не расходится с продом), подставляет эталонные данные из `fixtures/` и проверяет, что модель возвращает валидный JSON с обязательными ключами схемы (`name`, `contact`, `target_role`, `summary`, `experience[].bullets`, `education`, `skills`, `languages`; для `_match_prompt` дополнительно `ats_keywords`).

## Шаг 1. Проверить доступность Ollama

```powershell
Invoke-RestMethod "$(if ($env:OLLAMA_URL) { $env:OLLAMA_URL } else { 'http://localhost:11434' })/api/tags"
```

- В списке `models` должна быть модель из `OLLAMA_MODEL` (дефолт `qwen2.5:14b`). Если её нет — подсказать пользователю: `ollama pull qwen2.5:14b`.
- Если локальной Ollama нет (connection refused) — можно указать удалённый инстанс: `$env:OLLAMA_URL = 'http://host:11434'`. Без доступной Ollama прогон бессмыслен — все три кейса упадут с «Сервис генерации недоступен».

## Шаг 2. Запустить прогон

```powershell
# при необходимости переопределить модель/URL:
# $env:OLLAMA_URL = 'http://localhost:11434'
# $env:OLLAMA_MODEL = 'qwen2.5:14b'
python .claude/skills/prompt-eval/eval.py
```

**Внимание: на CPU прогон трёх промптов на 14B-модели может занять несколько минут.** Команде ставь щедрый таймаут (5–10 минут), не обрывай раньше времени.

## Шаг 3. Интерпретация результатов

По каждому промпту печатается `PASS`/`FAIL` и время ответа. Exit code 0 — все три прошли; 1 — есть хотя бы один FAIL. При FAIL выводится конкретная причина (не парсится JSON / какие ключи схемы отсутствуют) и первые 500 символов сырого ответа модели.

`3/3 PASS` — промпты и модель совместимы, можно мержить правку промптов или фиксировать смену модели. Времена ответа стоит сравнить с прошлым прогоном: резкий рост — повод проверить `num_predict`/нагрузку.

## Шаг 4. Если FAIL — типовые причины

- **Ответ не парсится как JSON** — модель добавила текст вокруг JSON или не следует инструкции «JSON ТОЛЬКО». Частое при смене модели на менее дисциплинированную. Решения: усилить инструкцию в промпте, вернуть прежнюю модель, либо рассмотреть `format: json` в запросе (сейчас `call_ai()` его не передаёт).
- **Нет ключей схемы** — схема в тексте промпта изменилась, а валидатор/фронтенд ожидают старую (или наоборот). Сверь блок «JSON ТОЛЬКО» в `main.py` со `SCHEMA` в `eval.py` — они должны описывать одно и то же.
- **Таймаут (504)** — `call_ai()` ждёт максимум 120 сек. На CPU 14B-модель может не успевать: уменьшить модель (`qwen2.5:7b`), прогреть её заранее (`ollama run <модель> ''`), или гнать на GPU-хосте через `OLLAMA_URL`.
- **Ответ обрезан (невалидный JSON в конце)** — упёрлись в `num_predict: 2048` в `call_ai()`. Проверь, не раздулись ли фикстуры или промпт.
