# Handoff: Редизайн «Резюмирую» (AI-генератор резюме)

Документ для разработчика (и для Claude Code), который будет переносить новый дизайн на
существующий сайт. Сайт построен на **Python + FastAPI**, фронтенд — **серверный HTML + CSS**
(шаблоны, без SPA-фреймворка). Этот документ самодостаточен: реализовать дизайн можно
только по нему, не видя исходного разговора.

---

## 1. Обзор

«Резюмирую» — сервис, который собирает резюме под конкретную вакансию с помощью AI.
Редизайн охватывает **три основных экрана** одного авторизованного веб-приложения:

1. **Мои резюме** (библиотека откликов) — стартовый экран с тремя режимами просмотра:
   Лента, Доска (канбан), Проводник.
2. **Создать резюме** (генерация) — пошаговая форма: вакансия → профиль → акценты → сборка.
3. **Редактор резюме** — результат генерации: AI-сводка изменений, разделы резюме,
   живой предпросмотр, экспорт в PDF, управление статусом отклика.

Все три экрана живут под общим «каркасом»: верхняя шапка, панель хлебных крошек и подвал.

Поддерживаются **две темы** — тёмная (по умолчанию) и светлая, переключаются кнопкой в шапке.

---

## 2. О файлах дизайна

В папке `design/` лежит **визуальный эталон**, созданный в HTML (`Резюмирую.dc.html`
+ рантайм `support.js`). Это **прототип, показывающий итоговый вид и поведение**, а
**не production-код для копирования напрямую**.

> Задача — **воссоздать этот дизайн в вашей среде** (FastAPI + Jinja2/шаблоны + CSS),
> используя ваши существующие паттерны: серверный рендеринг, CSS-файл проекта,
> прогрессивное улучшение через небольшие куски JS. HTML-эталон вставлять как есть не нужно —
> он написан на внутреннем шаблонизаторе прототипа. Берите из него **разметку, стили,
> значения и поведение**, а оформляйте по архитектуре вашего проекта.

Чтобы открыть эталон: откройте `design/Резюмирую.dc.html` в браузере (рядом должен лежать
`support.js`). Кликайте по навигации — все три экрана и обе темы интерактивны.

В папке `screenshots/` — снимки всех состояний (см. раздел 11).

## 3. Точность (fidelity)

**Hi-fi (высокая точность).** Это финальные цвета, типографика, отступы, радиусы,
тени и микровзаимодействия. Воссоздавайте UI **пиксель-в-пиксель**, используя точные
значения из раздела 4. Если в вашем проекте уже есть устоявшийся CSS-слой — приведите
эти токены к вашим переменным, но визуальный результат должен совпадать.

### 4.5. Радиусы, отступы, тени

- **Радиусы:** кнопки/инпуты `9–14px`; карточки `16–22px`; круглые элементы `50%`;
  пилюли/бейджи `8–999px`. Шкала по контексту: иконка-подложка `8–12px`,
  карточка вакансии `14–16px`, секция-контейнер `18–22px`.
- **Высоты контролов:** основные кнопки `40–50px`, компактные `32–38px`,
  инпуты `44px` (поиск) / `52px` (поле вакансии), квадратные иконки-кнопки `34–40px`.
- **Сетки:** макс. ширина контента `1240–1320px` (редактор и шапка-крошки шире — `1320px`),
  генерация — `1080px`. Боковые поля (gutter) `24px`. Зазоры в грид/флекс `10–20px`.
- **Шапка:** высота `62px`, `position: sticky; top:0`, `backdrop-filter: blur(16px)`.
- **Крошки:** липкая панель под шапкой, `position: sticky; top:62px`.
- **Липкие сайдбары** (правая сводка генерации, левая панель редактора, аэродром проводника):
  `position: sticky; top:112px`.

### 4.6. Иконки

Все иконки — **инлайн-SVG в стиле line / stroke** (`stroke-width` ≈ 2–2.4,
`stroke-linecap="round"`, `fill="none"`, `currentColor`), `viewBox="0 0 24 24"`,
размеры 13–20px. Никаких иконочных шрифтов. В нескольких местах для «настроения»
используются эмодзи (👋 ✨ ✦ 🌙 ☀️ и эмодзи разделов резюме 📇 ✍️ 💼 🛠️ 🎓) — их
можно заменить на line-иконки, если в проекте принято избегать эмодзи.

## 5. Общий каркас (на всех экранах)

### 5.1. Шапка (`<header>`, sticky, высота 62px)
- **Слева:** кнопка-логотип — квадрат 30×30 c радиусом 9px, заливка
  `linear-gradient(140deg,var(--accent-2),var(--accent))`, внутри SVG-иконка документа
  цветом `--accent-ink`; рядом слово **«Резюмирую»** (Manrope 800, 17px). Клик → Библиотека.
  (Текст логотипа скрывается на ширине ≤640px.)
- **По центру:** сегментированная навигация в капсуле (`--surface-2`, радиус 13px, padding 4px):
  кнопка **«Мои резюме»** (иконка списка) и кнопка-CTA **«Создать»** (иконка «+»).
  Активная «Мои резюме»/«Редактор» — фон `--surface` + тень `--shadow`.
  «Создать» в неактивном виде — `--accent-tint`/`--accent`; в активном (на экране генерации) —
  сплошной `--accent` + `--accent-ink`. Подписи кнопок скрываются на ширине ≤760px (остаются иконки).
- **Справа:** кнопка переключения темы (40×40, `--surface-2`, эмодзи 🌙/☀️) и
  круглый аватар 36×36 с инициалами «АИ» на градиенте.

### 5.2. Панель хлебных крошек (sticky, top:62px)
- Если не на стартовом экране — слева кнопка **«Назад»** (стрелка влево) + вертикальный разделитель.
- Далее крошки через разделитель-шеврон. Кликабельные — `--text-2` (hover → `--text`),
  текущая — `--text` 700, без действия.
- Пути:
  - Библиотека: `Мои резюме › <Лента|Доска|Проводник>`
  - Генерация: `Мои резюме › Новое резюме`
  - Редактор: `Мои резюме › <Компания> › <Название вакансии>`

### 5.3. Подвал (`<footer>`)
- Слева мини-логотип + «Резюмирую» / подпись «AI-резюме под каждую вакансию».
- Справа ссылки: **Тарифы · Публичная оферта · Политика конфиденциальности · Контакты**.
- Нижняя строка: `© 2025 Резюмирую.рф`, `--text-3`, 12.5px.

## 6. Экран «Мои резюме» (библиотека)

Контейнер `max-width:1240px; padding:34px 24px 90px`.

### 6.1. Приветствие
- Над заголовком — дата-метка цветом `--accent`, 13px/700 (напр. «Понедельник, 8 июня»).
- H1: «С возвращением, Алексей 👋» (32px/800, -0.03em).
- Подзаголовок (`--text-2`, 15px): сводка вида
  «8 резюме под 5 компаний · 2 на собеседовании · средний ATS 85%» (считается из данных).
- Справа — основная кнопка **«Создать резюме»** (высота 46px, `--accent`/`--accent-ink`,
  радиус 13px, тень `0 8px 22px var(--accent-tint)`, иконка «+»).

### 6.2. Карточки-статистика (грид 4 колонки, gap 14px)
Карточка: `--surface`, border `--border`, радиус 17px, padding 18px. Сверху строка
«иконка-подложка (26×26, `--accent-tint`/`--accent`) + подпись» (`--text-2`, 12.5px/600),
ниже — крупное число (30px/800) + дельта (`--accent`, 12px/600).

Четыре стата: **Активные отклики** (📨, значение = отклики со статусом ≠ draft, дельта «+3»),
**Собеседования** (🎯, кол-во `interview`, «эта неделя»), **Средний ATS** (⚡, среднее
по всем match-процентам + «%», «отлично»), **Офферы** (🏆, кол-во `offer`, «в работе»).

### 6.3. Тулбар
В одну строку (flex, wrap): поле **поиска** (44px, иконка-лупа слева, плейсхолдер
«Поиск по компаниям и вакансиям…», focus → border `--accent-line` + кольцо
`0 0 0 3px var(--accent-tint)`); ряд **фильтр-чипов** (скрыт в режиме «Доска»);
**переключатель режима** из 3 иконок-кнопок (Доска / Лента / Проводник) в капсуле
`--surface`. Активная иконка — `--accent-tint`/`--accent`.

**Фильтр-чипы:** Все / Черновики / Отправлено / Собес / Офферы. Каждый — пилюля с
счётчиком-бейджем. Активный — `--accent-tint` + border `--accent-line` + текст `--accent`,
бейдж `--accent`/`--accent-ink`. Неактивный — `--surface`/`--text-2`, бейдж `--surface-3`/`--text-3`.
Фильтр и поиск применяются ко всем трём режимам.

### 6.4. Режим «Лента» (по умолчанию)
Вертикальный список секций-компаний (gap 18px). Секция: `--surface`, радиус 22px, padding 18px.
- Шапка секции: аватар компании 42×42 (радиус 12px, цвет компании, белые инициалы),
  название (16.5px/700) + мета («N вакансий · средний ATS X%»), плашка «ATS X%»,
  кнопка **«+ Резюме»** (`--surface-2`).
- Внутри — авто-грид карточек вакансий (`repeat(auto-fill,minmax(270px,1fr))`, gap 12px).
  Карточка (`--surface-2`, радиус 16px): сверху бейдж-статус слева + кольцо ATS справа
  (круг 42×42, `conic-gradient(var(--accent) <match*3.6>deg, var(--ring-track) 0)`, в центре %),
  затем название вакансии (15px/700), зарплата · локация, и нижняя строка
  «<обновлено> … Открыть →». Hover карточки: border `--accent-line`,
  `translateY(-3px)`, тень `--shadow-lg`.

### 6.5. Режим «Доска» (канбан)
Горизонтальный грид колонок по статусам (`grid-auto-flow:column;
grid-auto-columns:minmax(280px,1fr); overflow-x:auto`). Колонка: `--surface-2`, радиус 18px,
padding 11px. Шапка колонки: цветная точка статуса (с halo `box-shadow:0 0 0 4px <bg>`) +
подпись + счётчик-бейдж. Карточки компактнее, чем в ленте (радиус 14px): строка
«аватар компании + название + мини-кольцо ATS 30×30», заголовок вакансии, низ
«зарплата · локация». Внизу колонки — пунктирная кнопка **«+ Добавить»**.

### 6.6. Режим «Проводник» (мастер-деталь)
Грид `300px 1fr`, gap 16px.
- **Слева** (`--surface`, радиус 20px, sticky top:112px): заголовок «Компании · N», затем
  список-строки компаний — аватар 34×34, название + «N · ATS X%», шеврон. Активная строка —
  `--accent-tint` + border `--accent-line`.
- **Справа** (`--surface`, радиус 22px): шапка выбранной компании (аватар 50×50, название
  20px/800, сайт · мета, кнопка **«+ Резюме под вакансию»** `--accent`), ниже — список
  вакансий строками: кольцо ATS 46×46, название + зарплата·локация·дата, плашка-статус, шеврон.
  Hover строки → фон `--surface-2`.

## 7. Экран «Создать резюме» (генерация)

Контейнер `max-width:1080px`. Сверху — пилюля-бейдж «✨ AI собирает резюме под конкретную
вакансию» (`--accent-tint`/`--accent`), H1 «Новое резюме под вакансию» (27px/800), подзаголовок.

Тело — грид `1fr 348px`, gap 18px.

### 7.1. Левая колонка — три шага
Каждый шаг — секция `--surface`, радиус 16px, padding 16px, с номерным бейджем
(28×28, `--accent`/`--accent-ink`, Manrope 800).

- **Шаг 1 — Вакансия.** Сегмент-переключатель «Ссылка / Текст вакансии / Файл».
  - *Ссылка:* инпут 52px с иконкой-цепочкой, плейсхолдер «hh.ru/vacancy/… или career.avito.com/…».
  - *Текст:* textarea (min-height 120px).
  - *Файл:* зона drag-&-drop (пунктир `--border-2`, иконка загрузки, «Перетащите PDF/скрин вакансии»).

## 9. Взаимодействия и поведение

- **Навигация между экранами** — через шапку, крошки и клики по карточкам вакансий
  (карточка/строка вакансии → Редактор по этой вакансии). «Назад» и логотип → Библиотека.
  При каждом переходе — `window.scrollTo(0,0)`.
- **Переключение темы** — кнопка в шапке ставит/снимает `data-theme="light"` на `<html>`.
  Сохранять в `localStorage`, восстанавливать при загрузке (чтобы не было «мигания» темы).
- **Поиск** — живая фильтрация по названию вакансии и названию компании (регистр игнор.),
  применяется во всех трёх режимах библиотеки.
- **Фильтр-чипы** — фильтрация по статусу; счётчики показывают кол-во по каждому статусу.
- **Переключатель режима** (Лента/Доска/Проводник) — меняет раскладку, сохраняя фильтр/поиск.
- **Дропдаун статуса в редакторе** — меняет статус отклика (отражается цветом/подписью).
- **Генерация** — кнопка «Собрать резюме» запускает сборку. В реальном приложении нужно
  **состояние загрузки** (≈30 сек по копирайту) и обработка ошибок; в прототипе переход мгновенный.

### Микровзаимодействия (hover/focus) — единый язык
Все переходы: `transition: transform .15s, box-shadow .15s, filter .15s` (карточки — .16–.18s).

| Состояние            | Эффект                                                                 |
|----------------------|------------------------------------------------------------------------|
| `hoverBtn`           | `border-color: --border-2; color: --text`                              |
| `hoverSoft`          | `border-color: --border-2; background: --surface-3`                    |
| `hoverPrimary`       | `filter: brightness(1.06); transform: translateY(-1px)`               |
| `hoverCard`          | `border-color: --accent-line; transform: translateY(-3px); box-shadow: --shadow-lg` |
| `hoverRow`           | `background: --surface-2; border-color: --border`                      |
| `focusInput`         | `border-color: --accent-line; box-shadow: 0 0 0 3px --accent-tint`     |

Анимации (`@keyframes`, есть в эталоне): `rsmspin` (вращение, для лоадеров),
`rsmpulse` (мигание прозрачности), `rsmrise` (появление снизу `translateY(8px)` + fade).
Кастомный скроллбар: ширина 11px, `thumb` = `--border-2` (hover → `--text-3`).
Выделение текста: `::selection { background: --accent-tint; color: --text }`.

## 10. Состояние и данные

Приложение работает поверх одной коллекции **компаний с вакансиями-откликами**. Минимальная
модель (на FastAPI — Pydantic-схемы / шаблонный контекст):

```
Company { id, name, initial, color, site, vacancies[] }
Vacancy { id, title, status(draft|sent|interview|offer), match(0..100),
          salary, loc, updated }
```

Состояние UI, которое надо отразить (через серверные параметры/маршруты или лёгкий JS):
`screen` (library|generate|editor), `theme`, `libView` (feed|board|explorer),
`filter` (all|draft|sent|interview|offer), `query` (поиск), `selectedCompany`,
`target` (выбранная вакансия для редактора), `editStatus`, `gen.{tab,profile,emphasis}`.

Производные значения (считать на сервере): средний ATS, кол-во по статусам, счётчики чипов,
угол conic-gradient для колец = `match * 3.6deg`, склонение «вакансия/вакансии/вакансий».

Демо-данные (5 компаний, 8 вакансий) лежат в эталоне (`Резюмирую.dc.html`, объект `data`) —
используйте их как фикстуры для верстки, реальные данные подставит бэкенд.

## 11. Ассеты

- **Шрифт:** Manrope (Google Fonts) — подключить `<link>` как в разделе 4.4 или
  раздать локально (`woff2`, веса 400–800).
- **Иконки:** все инлайн-SVG (line-стиль, `currentColor`) — в проекте можно вынести в
  набор/спрайт. Точные пути есть в эталоне.
- **Логотипы компаний:** в макете — цветные плашки с инициалами (не настоящие лого).
  При интеграции замените на реальные лого, сохранив размер/радиус подложки.
- **Изображений-растров нет.** Декоративные фоны — чистый CSS (радиальные градиенты + точки).
- **Эмодзи** — опциональны (см. 4.6).

## 12. Адаптивность (брейкпоинты из макета)

| Ширина     | Поведение                                                                         |
|------------|-----------------------------------------------------------------------------------|
| `≤960px`   | Статы в 2 колонки; гриды генерации/редактора/проводника — в 1 колонку; липкие сайдбары становятся статичными; кредиты скрыты. |
| `≤760px`   | В основной навигации остаются только иконки (подписи `data-navlabel` скрыты).      |
| `≤640px`   | Уже гутеры (16px); текст логотипа скрыт; статы 2 колонки с меньшим gap; карточки/профили/«бумага» в 1 колонку; заголовок приветствия 25px. |
| `≤400px`   | Приветствие 23px; кнопка «Скачать PDF» во всю ширину.                              |

Раскладка строится на CSS Grid/Flex с `gap` (не на отступах отдельных элементов) — это
упрощает адаптив и переупорядочивание.

### Рекомендация по интеграции в FastAPI
1. Заведите CSS-файл с токенами из раздела 4 (`:root` + `html[data-theme="light"]`).
2. Свёрстайте общий шаблон-каркас (шапка / крошки / подвал) и три шаблона экранов
   (Jinja2), подставляя данные из бэкенда.
3. Реализуйте режимы библиотеки и фильтры через query-параметры маршрута
   (`?view=board&filter=interview&q=...`) — серверный рендер, без SPA.
4. Тему, дропдаун статуса и живой поиск (если нужен мгновенный) добавьте небольшим
   ванильным JS поверх серверной разметки.
5. Сверяйтесь со скриншотами и открытым эталоном — целевая точность пиксель-в-пиксель.
